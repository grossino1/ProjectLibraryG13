Detailed Description

Gestisce l'insieme di tutti i prestiti attivi.

Questa classe agisce come contenitore principale per gli oggetti Prestito. Utilizza un TreeSet per garantire che non ci siano duplicati. Delega le regole di validazione (numero minimo di copie e numero massimo di prestiti per utente) alla classe GestorePrestito.

Invariant
elencoPrestiti != null (La struttura dati è sempre inizializzata).
gestore != null (La classe deve sempre avere un riferimento valido al gestore delle regole).

inline

Tenta di registrare un nuovo prestito nel sistema.

Il metodo divide l'operazione in quattro fasi:

  1. Delega al GestorePrestito la validazione dei dati.
  2. Se la validazione passa, crea l'oggetto Prestito e lo aggiunge al TreeSet.
  3. Delega al GestorePrestito l'aggiornamento dei dati.
  4. Salva l'elenco aggiornato sul file.
Precondition
isbn != null && !isbn.isEmpty()
matricola != null && !matricola.isEmpty()
Postcondition
Se successo: elencoPrestiti.size() == old_size + 1 e l'elenco prestiti aggiornato viene salvato sul file binario.
Se fallimento/errore: elencoPrestiti.size() == old_size
Parameters
[in]isbnIl codice ISBN del libro da prestare.
[in]matricolaLa matricola dell'utente che richiede il prestito.
