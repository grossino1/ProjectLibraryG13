Detailed Description

Gestisce l'insieme di tutti gli utenti registrati.

Author
chiara

Questa classe agisce come contenitore principale per gli oggetti Utente. Utilizza un TreeSet per garantire che gli utenti siano sempre ordinati (per cognome e nome, come definito in Utente.compareTo) e che non ci siano duplicati.

Invariant
listaUtenti != null (La struttura dati interna è sempre inizializzata).
listaUtenti non contiene valori null.
Author
chiara
Version
1.0

◆ registrazioneUtente()

void GestioneUtente.ListaUtenti.registrazioneUtente ( Utente u) throws ListaUtentiPienaException, MatricolaNotValidException, UtentePresenteException, IOException
inline

Registra un nuovo utente nel sistema.

Aggiunge un utente alla collezione. Se l'utente è già presente (stesso Cognome, Nome e Matricola), il Set non lo duplicherà (rispettando l'implementazione del compareTo() definito in Utente) e sarà lanciato un errore. Inoltre se la matricola non è valida strutturalmente verrà lanciato un altro tipo di errore.

Precondition
u != null (Non è possibile registrare utenti nulli).
Postcondition
listaUtenti.contains(u) == true.
size() >= old_size().
Parameters
[in]uL'oggetto Utente da registrare.
Exceptions
IllegalArgumentExceptionSe l'utente inserito come parametro è nullo.
ListaUtentiPienaExceptionSe la listaUtenti è piena.
MatricolaNotValidExceptionSe l'utente ha un formato di matricola non valido.
UtentePresenteExceptionSe l'utente passato come parametro è già presente all'interno della lista degli utenti.
IOExceptionSe si verifica un errore di input/output durante la scrittura sul file.
